# Sticker Printer Extension

一个用于衣云采购系统的 Chrome 扩展。

该扩展会在指定采购页面注入“刷新数据”和“打印贴纸”按钮，自动抓取页面接口返回的采购明细，生成贴纸并调用浏览器打印。

## 功能简介

- 在采购管理页面自动注入操作按钮
- 拦截采购相关接口数据并缓存当前页面结果
- 自动生成采购贴纸并调用浏览器打印
- 支持补充加工方信息
- 支持根据送货明细条数自动决定打印张数
- 支持悬浮按钮上下拖动，并记住拖动后的位置

## 适用页面

- 站点：`https://yy.singbada.cn`
- 页面路径前缀：`/purchasingManagement`

只有命中上述采购管理页面时，扩展按钮才会显示。

## 技术实现

- Chrome Extension Manifest V3
- `background.js`：后台注入与消息转发
- `content.js`：按钮、数据缓存、打印逻辑、条数逻辑、拖动逻辑
- `inject.js`：运行在页面上下文，负责拦截 XHR / fetch
- `style.css`：打印样式与悬浮按钮样式

## 安装方式

1. 打开 Chrome
2. 访问 `chrome://extensions`
3. 开启右上角“开发者模式”
4. 点击“加载已解压的扩展程序”
5. 选择当前项目目录

## 使用说明

1. 打开衣云采购管理页面
2. 页面右下区域会出现：
   - `刷新数据`
   - `打印贴纸`
3. 点击 `刷新数据`
   - 自动触发页面查询
   - 拦截当前页面采购数据并缓存
4. 点击 `打印贴纸`
   - 如未缓存数据，会先尝试刷新
   - 自动整理贴纸字段
   - 自动补充加工方
   - 自动计算送货条数
   - 调用浏览器打印

## 贴纸内容

当前贴纸会输出以下信息：

- 订单编号
- SKC
- 采购单号
- 货号
- 物料名称
- 颜色 / 类型
- 下单日期
- 加工方
- 送货数

## 条数打印逻辑

扩展会额外调用以下接口获取送货条数：

1. `/apc/purchase.purchaseOrder/index`
   - 使用 `old_order_id`
   - 使用 `purchase_sn`
   - 获取 `purchase_detail_id`
2. `/apc/purchase.purchaseOrder/selectDelivery`
   - 使用 `purchase_detail_id`
   - 汇总返回结果中的 `slip_num`

处理结果如下：

- 若获取到条数，则贴纸展示 `（共X条）`
- 若条数为 `X`，则打印 `X` 张贴纸
- 若未获取到条数，则保持原有打印逻辑，不展示 `（共X条）`

## 按钮拖动说明

- 支持按钮组上下拖动
- 长按后拖动即可移动位置
- 普通点击不会触发拖动
- 按钮位置会保存在本地，下次打开页面继续沿用
